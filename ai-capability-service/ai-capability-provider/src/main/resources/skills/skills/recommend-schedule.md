# 生成排班推荐

## 指令

当需要生成最终推荐时，请执行以下操作：

1. **确认数据完整性**：确保已通过工具获取了足够的医生排班数据。
2. **应用决策原则**：按照科室匹配、职称权重、负载均衡、时间段选择的原则进行排序。
3. **生成推荐结果**：输出 1-3 条最优推荐，包含具体的医生、时间段和推荐理由。
4. **格式化输出**：严格返回 JSON 格式，不包含 Markdown 代码块标记。

## 决策原则

### 1. 科室匹配度优先

- 优先选择科室与业务需求匹配的医生
- 内科需求优先选择内科医生
- 如果没有完全匹配，选择相关科室

### 2. 职称权重

```
主任医师 > 副主任医师 > 主治医师 > 住院医师
```

职称越高，处理复杂病例的能力越强，优先级越高。

### 3. 负载均衡

| 负载状态 | 待处理+进行中 | 推荐优先级 |
|---------|--------------|-----------|
| 轻载 | ≤ 3 | 高 |
| 适中 | 4-6 | 最高 ⭐ |
| 较重 | 7-10 | 中 |
| 过载 | > 10 | 低 |

**重要**：负载适中（4-6）的医生优先级最高，既保证效率又避免过载。

### 4. 时间段选择

- 优先选择 PENDING 状态的时间段
- 上午时段优先于下午时段（一般情况）
- 考虑患者的特殊时间需求

## 输出格式

严格返回 JSON，不包含 Markdown 格式标记：

```json
{
  "schedules": [
    {
      "id": 0,
      "doctorName": "张医生",
      "patientName": null,
      "schedule": "上午门诊",
      "scheduleCategory": null,
      "scheduleCategoryName": null,
      "scheduleDay": "2024-01-15",
      "priority": null,
      "priorityDesc": null,
      "status": null,
      "statusDesc": null,
      "result": null,
      "link": null,
      "startTime": "09:00",
      "endTime": "10:00"
    }
  ],
  "recommendation": "推荐张医生（主任医师，15年经验）负责该时段。理由：1) 科室匹配（内科）；2) 职称最高；3) 当日负载适中（5个待处理）；4) 上午时段患者等待时间短。"
}
```

## 推荐理由模板

```
推荐{医生姓名}（{职称}，{经验}年经验）负责该时段。

理由：
1) 科室匹配：{科室名称}，{匹配程度}
2) 职称优势：{职称}，{职称优势说明}
3) 经验丰富：{经验}年工作经验
4) 负载均衡：当日{数量}个待处理，{负载状态}
5) 时间段：{时间段}，{时间段优势说明}
```

## 异常情况处理

### 无符合条件的医生

```json
{
  "schedules": [],
  "recommendation": "当前日期没有符合条件的医生。建议：1) 调整预约日期；2) 放宽科室要求；3) 联系管理员确认排班情况。"
}
```

### 指定医生不可用

```json
{
  "schedules": [],
  "recommendation": "指定的医生（ID: 1001）在当日没有可用时段。建议：1) 选择其他日期；2) 选择同科室其他医生。"
}
```

## 质量检查清单

输出前请确认：

- [ ] 是否已调用工具获取真实数据？
- [ ] 推荐的医生是否在当日有排班？
- [ ] 推荐的时间段是否可用（PENDING 状态）？
- [ ] 推荐理由是否具体、有说服力？
- [ ] JSON 格式是否正确？
